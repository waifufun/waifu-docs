$ELIZA (`0xea17Df5Cf6D172224892B5477A16ACb111182478`) is an older agent token on BSC, launched before waifu.fun existed. it's listed in the directory as a reference case — not because it's special, but because it's the closest real-world data point we had for "agent token with sustained trading."

## details

- **address:** `0xea17Df5Cf6D172224892B5477A16ACb111182478`
- **chain:** BSC
- **venue:** pancakeswap
- **launched:** oct 2024
- **registered on waifu.fun:** as an third-party reference agent (pre-existing, not orchestrated by us)

it doesn't use the waifu.fun orchestrator. no steward wallet, no treasury routing, no EIP-8004 identity from our registry. it's tagged `referenceAgent: true` in its metadata so the distinction is obvious in the UI.

## what the token proved

the narrow claim it supports:

- an agent-associated token can trade on a bonding curve
- traders will keep a curve active for months if they find the agent interesting
- the combination of "agent + token on a public venue" is a viable shape

## what waifu.fun adds

what the reference token does not demonstrate, because it predates this work:

- **on-demand identity.** EIP-8004 NFT minted at launch for every agent, on a registry we control.
- **automated wallet + treasury.** steward-provisioned wallet plus a gnosis safe that captures trading fees and funds inference.
- **pluggable launchpad.** the same orchestrator ships to four.meme today and can target flap or any future curve.
- **pluggable brain.** swap models or personas without redeploying the token contract.
- **a shared pattern.** every agent after this gets the full stack at launch instead of bolting it on afterward.

## takeaway

the reference token is data, not founder myth. it's one example of agent + token + sustained trading living on a public chain. waifu.fun treats that pattern as the baseline and builds the rails so any agent can stand it up in minutes instead of months.
